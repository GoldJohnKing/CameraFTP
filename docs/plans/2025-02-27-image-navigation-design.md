# 内置预览图片导航功能设计

**日期:** 2025-02-27  
**功能:** Windows 平台内置预览窗口图片导航  
**状态:** 已批准，待实现

---

## 背景与目标

当前内置预览功能仅支持查看最新接收的图片，用户无法浏览存储路径下的其他图片。本功能添加图片导航能力，允许用户在预览窗口中切换到同一存储路径下的其他图片。

## 需求规格

### 1. 文件扫描与索引（启动时）

**触发时机:** 应用启动时（主窗口和预览窗口启动时）

**扫描范围:**
- 路径：`save_path` 配置目录及其所有子目录（递归）
- 文件扩展名：`.jpg`, `.jpeg`, `.heif`, `.hif`, `.heic`（不区分大小写）
- 排序依据：文件系统**修改时间**（mtime），从新到旧

**性能考虑:**
- 使用异步递归扫描（`tokio::fs::read_dir`）
- 扫描在后台线程执行，不阻塞 UI
- 扫描结果缓存到内存，后续导航直接从缓存读取

### 2. "最新照片"按钮行为变更

**场景 A: 从未接收过图片**
- 按钮显示扫描到的最新文件（修改时间最新）
- 文件名为完整文件名（不含路径）

**场景 B: 已接收过图片**
- 按钮始终显示最新接收到的文件（FTP 上传的文件）
- 即使该文件不是磁盘上最新的，也优先显示最近上传的

**技术实现:**
```rust
// 伪代码
fn get_latest_photo_button_text() -> String {
    if let Some(last_uploaded) = ftp_last_uploaded_file {
        // 场景 B: 有上传记录，显示最新上传
        last_uploaded
    } else {
        // 场景 A: 无上传记录，显示扫描到的最新文件
        scanned_files.first().map(|f| f.filename.clone()).unwrap_or("无".to_string())
    }
}
```

### 3. 文件列表实时维护

**更新时机:**
1. 启动时：完整扫描并建立列表
2. 接收到新文件时：将新文件插入列表头部（最新位置）
3. 删除文件时：从列表移除对应项

**数据结构:**
```rust
pub struct FileIndex {
    pub files: Vec<FileInfo>,  // 按修改时间排序（新→旧）
    pub current_index: Option<usize>,  // 当前预览的文件索引
}

pub struct FileInfo {
    pub path: PathBuf,      // 完整路径
    pub filename: String,   // 文件名（显示用）
    pub modified_time: SystemTime,
}
```

### 4. 导航按钮设计

**按钮布局**（底部工具栏从左到右）：

```
[文件名] [100%] | [← 上一张] [→ 下一张] [🏠 最新] [重置] [全屏] [置顶] [文件夹]
```

**按钮功能:**

| 按钮 | 图标 | 功能 | 禁用条件 |
|------|------|------|----------|
| 上一张 | ← | 切换到更旧的文件（索引+1） | 当前是最后一张 |
| 下一张 | → | 切换到更新的文件（索引-1） | 当前是第一张 |
| 最新 | 🏠 | 跳转到第一张（最新文件） | 已在第一张 |

**图标说明:**
- "最新"按钮使用**房子/主页图标**（🏠），表示"回到起点/最新"
- 或者使用**书签/旗帜图标**（🔖），表示"标记点"
- 或者使用**星星图标**（⭐），表示"收藏/精选"

**快捷键（可选）:**
- ← 方向键：上一张
- → 方向键：下一张
- Home 键：最新

### 5. 文件格式过滤规则

**支持的格式（严格匹配，不区分大小写）:**
- `.jpg`, `.jpeg`
- `.heif`, `.hif`, `.heic`

**过滤行为:**
- 扫描时：非上述格式的文件**跳过**，不加入列表
- FTP 上传时：非上述格式的文件**不显示**在"最新照片"按钮
- 自动预览：非上述格式的文件**不自动打开**预览窗口
- 手动打开：通过 StatsCard 点击打开时，**仅当格式匹配时才打开**

### 7. 状态管理

**后端状态:**
```rust
// 新增到 AutoOpenService 或独立 FileIndexService
pub struct FileIndexService {
    files: Arc<RwLock<Vec<FileInfo>>>,
    current_index: Arc<RwLock<Option<usize>>>,
    save_path: PathBuf,
}
```

**前端状态:**
- 预览窗口本地状态：`currentIndex`, `totalFiles`
- 可选：使用 Zustand store 共享文件列表（如果需要在主窗口显示缩略图列表）

### 8. 错误处理

| 场景 | 处理方案 |
|------|----------|
| 扫描时文件被删除 | 忽略错误，继续扫描其他文件 |
| 导航时文件被删除 | 显示错误提示，刷新列表 |
| 存储路径不存在 | 显示空列表，提示用户检查路径 |
| 文件无读取权限 | 跳过该文件，记录 warning 日志 |

### 9. 性能优化

1. **增量扫描:** 只扫描修改时间晚于上次扫描时间的文件（后续优化）
2. **延迟加载:** 预览窗口打开时才请求文件列表
3. **缓存元数据:** 文件信息（路径、时间）缓存，不缓存图片数据
4. **防抖导航:** 快速点击导航按钮时防抖处理
