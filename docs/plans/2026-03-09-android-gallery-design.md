# Android Gallery Page Design

Date: 2026-03-09
Status: Approved

## Overview

Add a gallery page for Android platform to display image thumbnails from the storage path using native Android MediaStore for optimal performance with large images (30-50MB).

## Requirements

- **Platform**: Android only
- **Layout**: Grid layout (3 columns, similar to photo album apps)
- **Interaction**: Tap image → open with third-party app via `PermissionAndroid.openImageWithChooser()`
- **Performance**: Lazy loading with Intersection Observer

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    Android Native Layer                  │
├─────────────────────────────────────────────────────────┤
│  GalleryBridge.kt                                       │
│  ├─ getGalleryImages(path): List<GalleryImage>          │
│  │   • Query images in specified directory via MediaStore│
│  │   • Get system-cached thumbnails (MINI_KIND: 512x384) │
│  │   • Return thumbnail base64 + file path               │
│  └─ Register in MainActivity                             │
└─────────────────────────────────────────────────────────┘
                           ↓ JS Bridge
┌─────────────────────────────────────────────────────────┐
│                    React Frontend Layer                  │
├─────────────────────────────────────────────────────────┤
│  GalleryCard.tsx                                        │
│  ├─ Call GalleryBridge.getGalleryImages()               │
│  ├─ Grid layout for thumbnails (CSS Grid)               │
│  ├─ Lazy loading: Intersection Observer                 │
│  └─ Tap → PermissionAndroid.openImageWithChooser()      │
└─────────────────────────────────────────────────────────┘
```

## Implementation Details

### 1. Android Native - GalleryBridge.kt

**Location**: `src-tauri/gen/android/app/src/main/java/com/cameraftp/app/bridges/GalleryBridge.kt`

**Responsibilities**:
- Query images in storage path using `MediaStore.Images.Media`
- Get cached thumbnails via `MediaStore.Images.Thumbnails.getThumbnail()`
- Convert thumbnails to base64
- Return JSON array of image data

**Data Structure**:
```json
{
  "images": [
    {
      "id": 12345,
      "path": "/storage/emulated/0/DCIM/CameraFTP/IMG_001.jpg",
      "filename": "IMG_001.jpg",
      "thumbnail": "data:image/jpeg;base64,/9j/4AAQ...",
      "dateModified": 1709500000000
    }
  ]
}
```

**Error Handling**:
- Return empty array if directory doesn't exist or no permission
- Log errors for debugging

### 2. Frontend - GalleryCard.tsx

**Location**: `src/components/GalleryCard.tsx`

**Features**:
- Only render on Android (check `window.GalleryBridge` existence)
- Load image list on mount
- CSS Grid 3-column layout
- Intersection Observer for lazy loading thumbnails
- Tap to open with `PermissionAndroid.openImageWithChooser()`
- Loading and empty states

### 3. Navigation Changes

**Files to modify**:
- `src/stores/configStore.ts` - Add `'gallery'` to `activeTab` type
- `src/components/BottomNav.tsx` - Add gallery tab button (use `Images` icon)
- `src/App.tsx` - Conditionally render `GalleryCard` when `activeTab === 'gallery'`

**Navigation structure**:
```
首页 (home) | 图库 (gallery) | 设置 (config)
```

### 4. Type Definitions

**Location**: `src/types/index.ts`

```typescript
interface GalleryImage {
  id: number;
  path: string;
  filename: string;
  thumbnail: string; // base64 data URL
  dateModified: number;
}
```

## Performance Considerations

1. **Thumbnail Caching**: Use Android's built-in MediaStore thumbnail cache (already generated by system)
2. **Lazy Loading**: Only render thumbnails in viewport
3. **Base64 Size**: MINI_KIND thumbnails are ~512x384, base64 size ~50-100KB each
4. **Memory**: RecyclerView-style virtualization not needed due to thumbnail caching
